## TeamX Automation Practical Assignment by Nkosinathi Jiyane

#### Login and Add New User flow on the TeamX Web Application using Selenium WebDriver (Java).

- This simple framework follows POM design patterns to support the reusability of objects across the framework.
- Please note on this test automation framework we did not use data driven approach for now, hence we hard coded the test data when we send the keys. Login is required to read from config files or json even or csv file.

### Tools Used

-  IntelliJ IDEA
-  Java
-  Maven
-  Selenium WebDriver (latest version)
-  Logger
-  GitHub / Git - Project repository.
-  Extent reporting

### Project access on Github

- Clone the project : https://github.com/Jiyanenm/TeamX_Automation_Test_.git

### Project Structure

- TeamX_Automation_Test_ : main module or project
- test - testing package
- java - java package
- base - Selenium base class to initialize or driver
- page - page objects with for our elements
- test - Test packages, it's our test scripts.

### How to run / Test

- Build command : mvn clean install
- Test command : mvn test
